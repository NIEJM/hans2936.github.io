使用Github Pages制作个人网站(上)：
网站框架

2017年10月20日 本文点击数:
进入专题：网页制作 进入评论栏

一直想做个博客网站，去网上查了一下，最先查出来的是云服务器+WordPress，不过又要买服务器，又要买域名，还得备案。
后来发现每个Github账户都可以开一个自己的静态网站（域名为 username.github.io），非常简单。
静态网站的定义参见维基百科-网页设计， 虽然方便，但是也有一些功能上的限制，最显著的特点就是不能运行php和asp，不能自动读写站内的文件。
不过静态网页依然可以实现个人博客需要的所有功能，而我也只是学习一下做网页、博客什么的，这个方案已经是最好的了。
这篇博客介绍我搭建网站框架的过程。 网站的进一步建设参加下篇：交互设计。

拥有并了解Github 账户

第一步 注册

没啥好说的，进入https://github.com，然后注册一个账户就可以了。

第二步 确保可以自由地创建/修改/下载/上传一个库（repository）

网站是由很多html，css，js等文件组成的，对github pages来说，一个repository就是存放这些文件的文件夹，所以确保可以随心所欲的管理一个库是最基本的。

这里是第一个官方教程：GitHub:Hello world。可以通过按钮来理解一个库的构造和维护，不过这只能让你创建修改一个Readme文件。
这里你会意识到每个库是可以创建分支(branch)的，实际操作中，如果对版本要求没那么严格（个人网站而已），你可以忽略branch的存在，只用master一个默认branch即可。

这个时候你已经可以用网页看到这么一个库了，这个图是本站的库，到这一步你的库可能只有一个Readme：

Github的关键就是可以用Git来控制（维基百科-Git），这是一个版本工具（我的理解是一个每次上传都备份一个版本，并且可以协调多人同时修改的公用网盘）。 当在确定自己的终端可以使用 git --version 命令（Mac+Xcode自带），并且安装了SSH-Key 之后，你就可以在本地新文件夹内使用如下的基本命令来控制你的库了：

git clone 你的SSH地址（下载你的reposity）
git pull （联系服务器，更新本地文件夹）
git add --all （写入本地的所有新文件）
git commit -am "随便说点啥" （在本地进行一次有记录地更新，包括本地所有文件的修改）
git push origin master （把本地的更新推送到服务器）

其中SSH地址可以用网页浏览的"Clone or Download"+"use SSH"按钮看到，比如说git@github.com:uername/reposity.git

SSH-Key决定了你的电脑有没有权限进行这些操作，这里有另一个官方教程：GitHub:SSH，这个是必须一步步跟下来的，Mac的重要命令包括：

ls -al ~/.ssh (查一下是不是已经有key了)
ssh-keygen -t rsa -b 4096 -C "your_email@example.com" （产生一个key，不单独设密码的话一路回车就行）
pbcopy < ~/.ssh/id_rsa.pub （把你的key复制到粘贴板，不这么复制可能太长了复制不下来）

Linux 和 Windows 进入刚才的教程看就行，如下图，在GitHub点击"右上头像"+"Settings"+"SSH and GPG Keys"，在这里添加了你刚刚复制的public Key之后，你就有权限进行上述的Git操作了。

搭建Github Page网站

第一步 创建一个Github Pages Repository

GitHub Pages repository跟普通的库是一样的，唯一的区别就是他的名字必须叫做username.gihub.io。这个官方教程 GitHub Pages 写的十分好懂，按这个做完之后你就有了一个你的网址 username.github.io，里面有一句 Hello World！

第二步 找个网站模版

现在离一个好网站只差一步了，就是把好网站服务器的所有文件拷贝到你的文件夹里面。
GitHub官方建议你使用博客生成工具 Jekyll，如果这样做的话，直接看官方教程：GitHub:Jekyll 就行了。在你安装好Jekyll之后，直接一条命令就可以产生一个网站文件夹， 然后进去编辑博客并发布就行了。GitHub在库的设置里面还提供Jekyll主题选择，也可以去网上搜主题或者直接拷贝别人的库。另外一个博客生成工具Hexo也差不多。
我当时并不了解博客工具，还希望有一个很炫的页面，所以并没有用博客工具，而是网站模版。我去了一个模版网站 HTML5 UP，然后下载了其中一个模版， git add 所有的文件到我的网站中。然后我直接本地打开了 index.html，就看到了网站预览，如图：

当你git push之后，你的域名就会变成这样一个网站了。这是一个朴素的模版，不过功能很全，便于管理。
不用博客工具的好处在于灵活（也能学到更多的网页知识）。坏处是写一个博客就多一个html，如果有比如超过50篇博客，对一个静态网站来说，批量管理（编辑他们的公共部分、给他们归类什么的）就麻烦了。
然而这个模版的公共部分并不复杂，对我这种习惯使用bash+vim（而且写不了几篇博客）的人来说，这是可以接受的。

第三步 开始发挥吧

这个模版有三个html，主页、素材页和博客页。这个模版的潜力基本上都显示在这三页了。
到目前为止框架是有了，但是网站里面的链接、按钮其实都是空的，先熟悉熟悉这个模版的html、css、js，然后就可以加入自己的修改了。
